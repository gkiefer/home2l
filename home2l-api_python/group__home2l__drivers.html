Define a new driver. 

Define a new event-based driver.

'drvName' is the unique local id (LID) of the new driver.
'(ERcState) successState' is the automatic reply sent out to the system
before 'func'is invoked. Typical values are:

    - 'rcsValid': everything ok (no further action by the driver necessary)
    - 'rcsBusy': potentially not ok; Driver must report a valid value later.

'data' is an optional reference to user data that will be passed
unchanged to any 'func' invocations.

'func' is a function to drive new values and will be called as follows:

    func (rc, vs, data)

where 'rc' is the resource to be driven and '(CRcValue) value' is the
new value to be driven.

For reporting values from the resource, the 'CResource::Report...'
methods have to be used.
